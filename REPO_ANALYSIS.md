## 🧠 Backend (AI Agent Logic)

### **Directory Structure**


### **Key Backend Files**

#### 1. **`Backend/Services/Agent_Utils.py`**
- **Purpose:** Core AI agent implementation
- **Main Class:** `Agent`
- **Constructor Parameters:**
  - `model_name` (default: "mistral-small-latest")
  - `temperature` (default: 0.5) - Controls response creativity
  - `agent_role` (default: "general") - Options: "General" or "Research"
- **Key Components:**
  - **LLM:** ChatMistralAI (via Mistral AI API)
  - **Embeddings:** MistralAIEmbeddings (for semantic search)
  - **Vector Store:** Chroma (in-memory or persistent storage)
  - **Text Splitter:** RecursiveCharacterTextSplitter (chunks: 4000 chars, overlap: 200)
  - **Memory:** InMemorySaver (conversation context)
- **Key Methods:**
  - `convert_and_store_to_vect_db()` - Process & store PDFs
  - `clear_vectorstore()` - Reset knowledge base
  - `Invoke_agent()` - Execute agent with streaming output
- **Features:**
  - Conditional model behavior for "Research" vs "General" roles
  - API key verification via `.env` file
  - Dynamic vector store path (Vercel `/tmp` or local `Backend/vector_datastore`)

#### 2. **`Backend/Services/MiddleWares.py`**
- **Purpose:** Custom middleware for request/response processing
- **Main Class:** `ResearcherMiddleware` (extends `AgentMiddleware`)
- **Functionality:**
  - Intercepts model calls before/after execution
  - Injects system prompts based on agent role & input type
  - **Two Agent Modes:**
    - **General Mode:** Provides detailed, comprehensive answers; anticipates user needs
    - **Research Mode:** (presumably deeper analysis, academic rigor)
  - **Tool Selection Logic (Strict Order):**
    1. `document_retrieval_tool` - HIGHEST PRIORITY if PDF/document present
    2. `general_search_mode` - For standard factual queries (no doc)
    3. `advance_search_mode` - For complex/technical queries (no doc)
    4. No tools - For greetings/small talk
  - **Citation Protocol:** Documents must include in-line citations for factual claims
  - **Key Feature:** If document is present, agent MUST use it (overrides search)

#### 3. **`Backend/Services/tools.py`**
- **Purpose:** Research toolkit with specialized search/retrieval functions
- **Main Class:** `ResearchToolkit` (extends `BaseToolkit`)
- **Tools Provided:**
  1. **`document_retrieval_tool(queries)`**
     - Semantic search within uploaded documents
     - Input: List of 1-3 search queries (batch processing)
     - Uses vector similarity search (k=5 results)
     - Returns concatenated raw text chunks separated by `\n\n`
     - Can be called multiple times or with batch queries
  2. **`exa_search`** - Integration with Exa API for web research
  3. **`duck_search`** - DuckDuckGo search fallback
- **Configuration:**
  - Requires `EXA_API_KEY` environment variable
  - Vector store configured via constructor
  - Pydantic config allows arbitrary types

### **Key Frontend Files**

#### 1. **`Frontend/server.py`** ⭐ PRIMARY INTERFACE
- **Purpose:** FastAPI backend server with REST API
- **Port:** 8000 (default)
- **CORS:** Fully open (all origins allowed)
- **Session State:** In-memory dictionary tracking:
  - Current model & mode
  - Uploaded file metadata
  - Agent instance cache
  - Vector store status
  
- **API Endpoints:**
  
  | Endpoint | Method | Purpose |
  |----------|--------|---------|
  | `/` | GET | Serve `index.html` (static frontend) |
  | `/api/config` | POST | Update model selection & agent mode |
  | `/api/upload` | POST | Upload PDF/image for context |
  | `/api/chat` | POST | Send message & stream response via SSE |
  | `/api/clear-file` | POST | Remove uploaded file & clear vector store |
  | `/api/clear` | POST | Reset entire conversation & agent state |
  | `/api/status` | GET | Get current session status |
  
- **Response Streaming:** `/api/chat` uses Server-Sent Events (SSE) for real-time streaming
  - Format: JSON chunks with `type` ("chunk", "done", "error") & `content`
  - Handles errors gracefully with error payloads
  
- **Model Handling:**
  - Research mode forces `magistral-medium-latest` model
  - General mode supports multiple Mistral variants
  
- **File Processing:**
  - PDF: Auto-convert & store in vector DB
  - Image: Metadata only (for future processing)
  - Other: Treated as text
  
- **Key Features:**
  - Lazy agent initialization (creates on first use)
  - Agent recreation on model/mode change
  - Automatic vectorstore cleanup on file clear

## 📝 Summary for Colleagues

**What is this?**  
A full-stack AI research assistant that helps students/researchers study topics by combining web search, document analysis, and AI reasoning powered by Mistral AI.

**How does it work?**  
1. User uploads documents (PDFs) or asks questions
2. FastAPI backend routes to AI agent
3. Agent uses tools (semantic search, web search) to find answers
4. Responses stream in real-time via Server-Sent Events
5. All deployed on Vercel (no server management)

**Key Features:**  
- Two modes: General (broad info) & Research (deep analysis)
- Document context (upload PDFs for specific answers)
- Web search integration (DuckDuckGo, Exa API)
- Real-time streaming responses
- Multiple LLM model options (Mistral)

**Tech Stack:** Python 3.12+, FastAPI, LangChain, Mistral AI, Chroma, Vercel
